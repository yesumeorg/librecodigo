Sistema de Programación Visual

v0.1.0-alpha

Conceptos

BlockCodigo es un sistema de programación visual que convierte bloques arrastrables en código funcional. Se basa en cuatro pilares fundamentales:

IR (Intermediate Representation)

Representación intermedia que almacena la estructura lógica de los bloques antes de convertirla en código.

Blocks

Elementos visuales arrastrables que representan sentencias, expresiones y eventos del programa.

Converters

Módulos que transforman código fuente en bloques visuales y viceversa.

Emitters

Generadores de código finales que toman el IR y producen código en lenguajes objetivo.

API de Bloques

Registra bloques personalizados con la API de BlockCodigo.registerBlock():

javascript 
BlockCodigo.registerBlock({
  id: 'print',
  label: 'Print to Console',
  category: 'io',
  shape: 'stack',
  ports: [{
    id: 'msg',
    type: 'string',
    direction: 'input'
  }],
  toCode: (block) => `console.log(${block.getPort('msg')});`
});

Ejemplo Avanzado: Bloque If/Else

javascript 
BlockCodigo.registerBlock({
  id: 'if_else',
  label: 'Si %condición entonces',
  category: 'logic',
  shape: 'c-block',
  ports: [
    { id: 'condition', type: 'boolean', direction: 'input' },
    { id: 'then', type: 'statement', direction: 'body' },
    { id: 'else', type: 'statement', direction: 'body' }
  ],
  toCode: (block) => {
    const cond = block.getPort('condition');
    const thenBody = block.getBody('then');
    const elseBody = block.getBody('else');
    return `if (${cond}) {\n${thenBody}\n} else {\n${elseBody}\n}`;
  }
});

Métodos del Bloque

javascript 
// Dentro de la función toCode(block):

// Obtener valor de un puerto de entrada
block.getPort('msg');          // → "Hola" o表达式

// Obtener cuerpo de un C-block
block.getBody('then');          // → código generado del cuerpo

// Obtener valor de un dropdown
block.getDropdown('op');       // → "+"

// Verificar si un puerto tiene conexión
block.hasPort('condition');    // → true/false

// Obtener la posición del bloque en el workspace
block.getPosition();             // → { x: 120, y: 80 }

Conversión Code ↔ Blocks

BlockCodigo soporta conversión bidireccional: puedes importar código existente como bloques visuales, y exportar bloques como código.

Código → Bloques (Importación)

El parser analiza el código fuente y genera un AST (Abstract Syntax Tree). Los Converters mapean cada nodo del AST a un bloque visual correspondiente.

javascript 
// Importar código al editor de bloques
BlockCodigo.importCode(sourceCode, {
  language: 'javascript',
  onImport: (blocks) => {
    // blocks contiene el array de bloques parseados
    console.log(`Importados ${blocks.length} bloques`);
  },
  onError: (err) => {
    console.error('Error de parseo:', err.message);
  }
});

Bloques → Código (Exportación)

Cada bloque ejecuta su función toCode(). El Emitter recorre el árbol de bloques y genera el código final respetando la indentación y sintaxis del lenguaje objetivo.

javascript 
// Exportar bloques a código
const code = BlockCodigo.exportCode({
  language: 'python',
  indent: 4,
  target: 'python3.11'
});

// resultado: código Python generado
console.log(code);
// → "print('Hola Mundo')"
// → "for i in range(10):"
// → "    print(i)"
La conversión no es 100% perfecta en ambos sentidos. Algunas construcciones complejas de código pueden no tener un equivalente visual directo. BlockCodigo marcará estos casos con advertencias.

Compiladores Soportados

BlockCodigo incluye emitters para los siguientes lenguajes:

LenguajeSoporteNotas
JavaScript Completo Soporte completo para ES2024+. Incluye async/await, destructuring y más.
Python Completo Python 3.8+. Soporta comprensiones, f-strings y type hints.
Batch Parcial Limitado por la naturaleza de Batch. Soporta comandos básicos, if/for.

Emitir código en diferentes lenguajes

javascript 
// Definir un emitter personalizado
BlockCodigo.registerEmitter('rust', {
  name: 'Rust',
  ext: '.rs',
  emit: (ir) => {
    const emitter = new RustEmitter();
    return emitter.generate(ir);
  }
});
Puedes crear emitters personalizados para cualquier lenguaje. Consulta la sección de API de Bloques para más detalles.

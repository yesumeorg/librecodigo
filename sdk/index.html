SDK para Plugins de Libre Código

Crea plugins potentes para extender las funcionalidades de Libre Código

v0.1.0-alpha

Introducción

El SDK para Plugins de Libre Código te permite crear extensiones que añadan nuevas funcionalidades al editor, como comandos personalizados, paneles laterales, bloques visuales y más.

Para empezar a desarrollar plugins necesitas:

  • Un editor compatible con Libre Código (v0.1.0 o superior)
  • Node.js v18+ (para herramientas de desarrollo)
  • Conocimientos básicos de JavaScript
Puedes instalar el CLI de desarrollo con npm install -g @libre-codigo/cli y crear un proyecto base con lc-plugin init mi-plugin.

Instalación Rápida

bash 
# Instalar el CLI
npm install -g @libre-codigo/cli

# Crear un nuevo plugin
lc-plugin init mi-plugin

# Iniciar modo desarrollo
cd mi-plugin
lc-plugin dev

Estructura de un Plugin

Cada plugin de Libre Código sigue una estructura de archivos simple y predecible:

mi-plugin/
├── manifest.json
├── index.js
└── styles.css
  • manifest.json — Metadatos del plugin: nombre, versión, permisos, etc.
  • index.js — Código principal del plugin. Se ejecuta al cargar.
  • styles.css — Estilos CSS opcionales para la UI del plugin.
También puedes incluir una carpeta assets/ para imágenes, fuentes y otros recursos estáticos.

manifest.json

El archivo manifest.json define la identidad y configuración de tu plugin. Es obligatorio y debe estar en la raíz del proyecto.

Campos

CampoTipoDescripción
namestringNombre único del plugin (kebab-case)
versionstringVersión semántica (ej: "1.0.0")
authorstringNombre del autor o equipo
descriptionstringDescripción breve del plugin
mainstringRuta al archivo de entrada (default: "index.js")
permissionsarrayPermisos requeridos (ej: ["fs", "network"])
dependenciesobjectDependencias de otros plugins

Ejemplo

json 
{
  "name": "json-formatter",
  "version": "1.0.0",
  "author": "Tu Nombre",
  "description": "Formatea JSON con un solo clic",
  "main": "index.js",
  "permissions": ["editor"],
  "dependencies": {}
}

API Disponible

El SDK expone un objeto global libre_codigo con los siguientes métodos:

MétodoDescripción
registerCommand(name, callback) Registra un comando que aparecerá en la paleta de comandos (Ctrl+Shift+P)
registerPanel(id, options) Registra un panel lateral con contenido personalizado
registerBlock(def) Registra un bloque visual para BlockCodigo
getEditor() Retorna la instancia actual del editor (CodeMirror)
readFile(path) Lee el contenido de un archivo del proyecto (requiere permiso "fs")
writeFile(path, content) Escribe contenido a un archivo del proyecto (requiere permiso "fs")
onEvent(name, callback) Escucha eventos del editor: save, open, change, etc.
showNotification(msg, type) Muestra una notificación: info, success, warning, error

Ejemplo de Uso

javascript 
// Registrar un comando
libre_codigo.registerCommand('format-json', () => {
  const editor = libre_codigo.getEditor();
  const code = editor.getValue();

  try {
    const formatted = JSON.stringify(JSON.parse(code), null, 2);
    editor.setValue(formatted);
    libre_codigo.showNotification('JSON formateado correctamente', 'success');
  } catch (e) {
    libre_codigo.showNotification('Error: JSON inválido', 'error');
  }
});

// Escuchar eventos
libre_codigo.onEvent('save', (file) => {
  console.log('Archivo guardado:', file.path);
});

Ejemplo Completo

A continuación se muestra el código completo de un plugin formateador de JSON:

index.js

javascript 
/**
 * JSON Formatter Plugin for Libre Código
 * Formatea y valida JSON directamente desde el editor
 */

// Registrar el comando principal
libre_codigo.registerCommand('json-formatter:format', () => {
  const editor = libre_codigo.getEditor();
  const cursor = editor.getCursor();
  const selection = editor.getSelection();

  let text = selection || editor.getValue();

  try {
    const parsed = JSON.parse(text);
    const formatted = JSON.stringify(parsed, null, 2);

    if (selection) {
      editor.replaceSelection(formatted);
    } else {
      editor.setValue(formatted);
    }

    editor.setCursor(cursor);
    libre_codigo.showNotification('✓ JSON formateado', 'success');
  } catch (e) {
    libre_codigo.showNotification(
      `Error en línea ${e.lineNumber}: ${e.message}`,
      'error'
    );
  }
});

// Registrar segundo comando: comprimir JSON
libre_codigo.registerCommand('json-formatter:minify', () => {
  const editor = libre_codigo.getEditor();
  const text = editor.getValue();

  try {
    const parsed = JSON.parse(text);
    editor.setValue(JSON.stringify(parsed));
    libre_codigo.showNotification('JSON comprimido', 'success');
  } catch (e) {
    libre_codigo.showNotification('JSON inválido', 'error');
  }
});

// Registrar panel lateral con vista previa
libre_codigo.registerPanel('json-preview', {
  title: 'Vista Previa JSON',
  icon: 'braces',
  render: (container) => {
    const update = () => {
      const editor = libre_codigo.getEditor();
      try {
        const data = JSON.parse(editor.getValue());
        container.innerHTML = `<pre class="json-tree">${
          JSON.stringify(data, null, 2)
        }</pre>`;
      } catch {
        container.innerHTML = '<p class="error">JSON inválido</p>';
      }
    };
    update();
    libre_codigo.onEvent('change', update);
  }
});

console.log('[json-formatter] Plugin cargado');

Publicación

Una vez que tu plugin esté listo, puedes publicarlo en el foro de plugins de Libre Código para que otros usuarios lo instalen.

  1. Verifica tu plugin

    Ejecuta lc-plugin validate para comprobar que tu plugin cumple todos los requisitos del SDK.

  2. Genera el paquete

    Ejecuta lc-plugin build para crear un archivo .lcplugin listo para distribuir.

  3. Sube al foro

    Crea un nuevo post en la sección Plugins del foro de Libre Código. Incluye: nombre, descripción, capturas y el archivo .lcplugin.

  4. Mantenimiento

    Responde a los usuarios, corrige bugs y actualiza la versión según el feedback recibido.

Asegúrate de no incluir datos sensibles ni claves API en tu plugin. Todos los plugins son revisados antes de ser publicados.

Preguntas Frecuentes

¿Puedo usar TypeScript en mi plugin?
Sí. Puedes escribir tu plugin en TypeScript y usar lc-plugin build para compilarlo a JavaScript. Asegúrate de incluir un tsconfig.json en la raíz de tu proyecto.
¿Cuál es el tamaño máximo de un plugin?
El tamaño máximo recomendado es de 2 MB para el archivo .lcplugin. Si necesitas incluir assets grandes, considera cargarlos bajo demanda desde una URL.
¿Los plugins tienen acceso al sistema de archivos?
Solo si declaras el permiso "fs" en tu manifest.json. El acceso está restringido a la carpeta del proyecto actual. No se permite acceso a rutas del sistema.
¿Puedo registrar múltiples comandos?
Sí. No hay límite en el número de comandos que puedes registrar. Cada comando debe tener un nombre único con formato plugin:comando.
¿Cómo depuro mi plugin?
Ejecuta lc-plugin dev para iniciar el modo desarrollo. Esto abre el editor con tu plugin cargado y habilita el inspector de plugins (F12) para ver logs y errores en tiempo real.
¿Puedo dependeer de otros plugins?
Sí. Declara las dependencias en manifest.json bajo el campo dependencies. El SDK se encargará de verificar que estén instalados antes de cargar tu plugin.
